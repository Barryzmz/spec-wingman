## Scope

### In Scope

- 員工線上請假申請（特休、病假、事假、喪假）
- 主管線上審核（同意/拒絕）
- 代理主管機制（並行審核，先審先得）
- 假別額度管理（預扣機制，到職周年日重置）
- 催辦通知（逾期 3 天未審核，Email + 站內訊息）
- 審核結果通知（Email + 站內訊息）
- HR 查看全公司請假紀錄
- HR 匯出 Excel 報表（部門/日期篩選）
- HR 查看代理主管設定
- 外部 HR 系統 API 整合（身份驗證 + 員工資料）

### Out of Scope

- 銷假功能（已核准假單取消）— 排入第二期（BR-019, NGOAL-001）
- 公司行事曆 / 國定假日管理（DEC-007）
- 組織架構管理（DEC-009，由外部 HR 系統提供）
- 多租戶架構（ASM-001，Pending）
- 多語系支援（ASM-003，Pending）

## Product Vision

將現行以 Line 群組口頭溝通的請假流程，轉換為 Web 線上系統，讓員工可自助申請、主管可線上審核、HR 可統一管理與匯出紀錄，解決遺漏與計算錯誤的問題。

## Workflow Requirements

| ID | From State | Action | To State | Actor | Side Effects |
| --- | --- | --- | --- | --- | --- |
| WR-001 | （新建） | 員工送出申請 | 待審核 | 員工 | 預扣額度 |
| WR-002 | 待審核 | 主管/代理同意 | 已核准 | 主管/代理 | 通知申請人（Email + 站內） |
| WR-003 | 待審核 | 主管/代理拒絕（附原因） | 已拒絕 | 主管/代理 | 歸還額度 + 通知申請人 |
| WR-004 | 待審核 | 員工撤回 | 已撤回 | 員工 | 歸還額度 |
| WR-005 | 待審核（>3天） | 排程 0:00/12:00 | 待審核 | 系統 | 催辦通知（Email + 站內） |
| WR-006 | → 已核准 | 狀態變更觸發 | — | 系統 | Email + 站內訊息通知申請人 |
| WR-007 | → 已拒絕 | 狀態變更觸發 | — | 系統 | Email + 站內訊息通知申請人（含原因） |

### State Machine

```
    [新建] ──預扣額度──→ [待審核]
                            │
              ┌─────────────┼─────────────┐
              ▼             ▼             ▼
          [已核准]      [已拒絕]       [已撤回]
         (通知申請人)  (歸還額度,     (歸還額度)
                       通知申請人)
              
    ※ 代理期間：原主管＋代理主管並行，先審先得
    ※ 已核准/已拒絕/已撤回 = 終態
    ※ 待審核 >3天 → 催辦通知（不改變狀態）
```

## Domain Model

### Entities

| Entity ID | 名稱 | 描述 |
| --- | --- | --- |
| ENT-001 | Employee | 系統使用者，基本資料由 HR API 提供 |
| ENT-002 | Manager | 員工直屬主管，隸屬關係由 HR API 提供 |
| ENT-003 | LeaveRequest | 一筆請假申請（含狀態、預扣額度） |

| ENT-008 | Notification | 站內訊息（催辦/審核結果） |
| ENT-009 | AnnualLeaveRule | 特休年資對照規則 |
| ENT-010 | BereavementLeaveRule | 喪假親等對照規則 |

### Relationships

```
Employee (from HR API)
  ├── 1:N ──→ LeaveRequest
  │              ├── N:1 ──→ LeaveType
  │              ├── N:1 ──→ Manager (reviewer)
  │              ├── 1:N ──→ Attachment (optional)
  │              └── 1:N ──→ Notification
  ├── 1:N ──→ LeaveBalance
  │              ├── N:1 ──→ AnnualLeaveRule
  │              └── N:1 ──→ BereavementLeaveRule
  └── N:1 ──→ Manager (supervisor, from HR API)
                 └── 1:0..1 ──→ ActingManager
```

## Glossary

| Term | 定義 |
| --- | --- |
| 特休 | 年度特別休假，依年資給予，到職周年日一次給足 |
| 病假 | 因疾病或傷害申請，年度 30 天 |
| 事假 | 因個人事務申請，年度 14 天 |
| 喪假 | 因親屬過世申請，依親等 3~8 天，100 天內使用 |
| 假單 | 一筆請假申請紀錄（leave request） |
| 審核 | 主管對假單同意或拒絕 |
| 催辦通知 | 逾期 3 天未審核的系統自動提醒 |
| 代理主管 | 主管指定的臨時代理人，並行審核先審先得 |
| 銷假 | 已核准假單的取消流程（第二期） |
| 預扣機制 | 送出即佔額度，可用 = 總額 - 已核准 - 待審核預扣 |
| 親等 | 親屬關係遠近分類，決定喪假天數 |
| 診斷書 | 醫療證明，病假 ≥ 3 天時必要；限 PNG/JPG/PDF，≤5MB |
