## Scope

### In Scope

- 員工線上提交請假申請（含假別、時間範圍、原因、代理人）
- 主管在系統上審核請假申請（核准 / 拒絕）
- 系統自動催辦逾期未審申請
- 員工自行取消待審核申請
- 主管指定代理人及代理期間
- 無上層主管員工申請時自動核准
- HR 查閱所有員工請假紀錄
- HR 匯出 Excel 格式請假報表（含篩選，篩選條件待確認 Q-017）
- HR 建立與維護員工基本資料

### Out of Scope

- 薪資計算或扣薪邏輯（GOAL non-target）
- 排班或班表管理（GOAL non-target）
- 非功能需求（效能、安全性、可用性等，來源未提及）

---

## Product Vision

取代目前以 Line 群組通知主管、主管再口頭通知 HR 的請假流程，建立一套線上請假管理系統，讓員工可以線上申請請假、主管可以在系統上審核、HR 可以查閱所有紀錄並匯出報表，消除人工傳遞造成的遺漏與計算錯誤。

**來源**：FACT-002, FACT-003, FACT-004, FACT-005, FACT-006, SRC-001

## Functional Requirements

### FR-001：員工提交請假申請

| Field | Value |
| --- | --- |
| Actor | 員工（ROLE-001） |
| Trigger | 員工欲申請請假 |
| Preconditions | 員工已登入系統 |
| Description | 員工填寫請假資訊並提交。申請時數以工作日計算（排除週末與國定假日），精度為半小時；申請時須指定代理人 |
| Source | FACT-013, DEC-003, DEC-011, DEC-013, DEC-019, DEC-020, SRC-001 |
| Status | Confirmed |

**Acceptance Criteria**

| AC ID | Given | When | Then |
| --- | --- | --- | --- |
| AC-001 | 員工已登入系統 | 填寫開始日期時間、結束日期時間（精度半小時）、假別、原因、代理人後提交 | 系統依工作日計算時數，建立申請單，狀態設為「待審核」 |
| AC-001b | 員工未選擇代理人 | 員工提交申請 | 系統阻止提交並提示代理人為必填 |
| AC-001c | 系統中該員工無指定上層主管 | 員工完成提交 | 申請單自動設為「已核准」，不進入待審核流程 |
| AC-002 | 申請時數超過該假別剩餘配額 | 員工提交申請 | 系統阻止提交並顯示剩餘配額不足 |
| AC-003 | 假別為病假，請假時數 > 24 小時 | 員工提交申請 | 系統要求上傳診斷書，未上傳時阻止提交 |
| AC-004 | 假別為病假，請假時數 ≤ 24 小時 | 員工提交申請 | 診斷書為選填，可不上傳即提交 |

### FR-002：主管審核請假申請

| Field | Value |
| --- | --- |
| Actor | 主管（ROLE-002）或代理主管（ROLE-004） |
| Trigger | 主管轄下員工有待審核申請 |
| Preconditions | 申請單狀態為「待審核」；申請員工屬於審核者管轄範圍 |
| Description | 主管查看待審核申請，執行核准或填寫原因後拒絕 |
| Source | FACT-015, FACT-016, DEC-005, DEC-006, SRC-001 |
| Status | Confirmed |

**Acceptance Criteria**

| AC ID | Given | When | Then |
| --- | --- | --- | --- |
| AC-005 | 主管查看待審核申請單 | 主管選擇核准 | 申請單狀態更新為「已核准」 |
| AC-006 | 主管查看待審核申請單 | 主管填寫拒絕原因後拒絕 | 申請單狀態更新為「已拒絕」，原因記錄於申請單 |
| AC-007 | 主管選擇拒絕但未填寫原因 | 主管提交審核結果 | 系統阻止提交並提示必須填寫拒絕原因 |
