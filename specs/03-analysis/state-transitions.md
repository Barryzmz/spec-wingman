# State Transitions

## 假單（LeaveRequest）狀態機

### 狀態定義

| 狀態 | 描述 | 來源 |
| --- | --- | --- |
| 待審核 | 假單已送出，等待主管或代理主管處理；額度已預扣 | SRC-001, WR-001, BR-025 |
| 已核准 | 主管或代理主管已同意 | SRC-001, WR-002 |
| 已拒絕 | 主管或代理主管已拒絕（含拒絕原因）；預扣額度已歸還 | SRC-001, WR-003, BR-027 |
| 已撤回 | 員工主動撤回；預扣額度已歸還 | SRC-001, WR-004, BR-027 |

### 狀態轉換

| State Flow ID | 實體 | 起始狀態 | 事件 | 目標狀態 | 副作用 | 關聯需求 |
| --- | --- | --- | --- | --- | --- | --- |
| ST-001 | LeaveRequest | （新建） | 員工送出請假申請 | 待審核 | 預扣假別額度（BR-025）；指派審核主管（代理期間原主管與代理主管並行，BR-028） | WR-001, FR-001, BR-025 |
| ST-002 | LeaveRequest | 待審核 | 主管/代理主管同意（先審先得） | 已核准 | 發送 Email + 站內訊息通知申請人 | WR-002, FR-010, WR-006, BR-028 |
| ST-003 | LeaveRequest | 待審核 | 主管/代理主管拒絕（附原因，先審先得） | 已拒絕 | 歸還預扣額度（BR-027）；發送 Email + 站內訊息通知申請人（含原因） | WR-003, FR-010, WR-007, BR-017, BR-027 |
| ST-004 | LeaveRequest | 待審核 | 員工撤回 | 已撤回 | 歸還預扣額度（BR-027） | WR-004, FR-005, BR-027 |
| ST-005 | LeaveRequest | 待審核（>3天） | 系統排程觸發（0:00/12:00） | 待審核 | 發送催辦 Email + 站內訊息給審核主管 | WR-005, FR-004, NFR-003 |

### 額度計算規則

```
可用額度 = 總額度 - 已核准天數 - 待審核預扣天數

假單歸屬年度 = 請假起始日所在的額度年度（BR-029）
額度重置不追溯待審核假單（V1 簡化做法）
```

      │ 預扣額度  │     │                               │

      │           │     ├── 主管/代理拒絕 ──→ [已拒絕]   │

      │           │     │                    通知申請人) │
      │           │     │                               │

### 約束條件

- 只有「待審核」狀態可轉換至其他狀態（WR-002, WR-003, WR-004）
- 「已核准」「已拒絕」「已撤回」為終態，第一期不可再變更（BR-019）
- 拒絕時必須附帶原因（BR-017）
- 代理主管的審核等同主管審核，結果為最終（BR-026）
- 代理期間原主管與代理主管並行可審核，先審先得（BR-028）
- 額度採預扣機制，送出即佔額度，拒絕/撤回歸還（BR-025, BR-027）
- 假單按請假起始日歸屬年度，額度重置不追溯待審核假單（BR-029）
