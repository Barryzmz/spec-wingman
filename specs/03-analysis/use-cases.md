## UC-001: 員工申請請假

- 主要角色：員工（ROLE-001）
- 前置條件：員工已透過 HR API 完成登入驗證（NFR-004）
- 觸發條件：員工選擇申請請假
- 基本流程：
  1. 員工選擇假別（特休 / 病假 / 事假 / 喪假）
  2. 員工填寫開始日期、結束日期、原因
  3. 系統計算請假天數（僅計算工作日，BR-024），檢查是否符合最小單位 30 分鐘（BR-014）
  4. 系統檢查該假別剩餘額度是否足夠（BR-025）
  5. 若假別為病假且天數 >= 3 天，系統要求上傳診斷書（BR-015）
  6. 員工送出申請
  7. 系統建立假單，狀態為「待審核」（WR-001）
  8. 假單指派給員工的直屬主管（或代理主管，若有設定）
- 替代流程：
  - 4a. 額度不足：系統阻擋送出並提示額度不足（BR-025）
  - 5a. 病假 < 3 天：不強制上傳診斷書（BR-016）
  - 8a. 主管已設定代理人且目前在代理期間內：假單指派給代理主管（FR-008）
- 後置條件：假單已建立並進入待審核狀態
- 關聯需求：FR-001, FR-002, BR-014, BR-015, BR-016, BR-023, BR-024, BR-025

## UC-002: 主管審核假單

- 主要角色：主管（ROLE-002）
- 前置條件：存在狀態為「待審核」的假單，且屬於該主管的下屬
- 觸發條件：主管開啟待審核假單
- 基本流程：
  1. 主管查看假單詳情（假別、日期、原因、附件）
  2. 主管選擇「同意」或「拒絕」
  3. 若選擇同意：假單狀態變更為「已核准」（WR-002）
  4. 系統透過 Email 與站內訊息通知申請人審核結果（FR-010, WR-006）
- 替代流程：
  - 2a. 主管選擇「拒絕」：必須填寫拒絕原因（BR-017）
  - 2a-1. 未填寫原因：系統阻擋送出
  - 2a-2. 已填寫原因：假單狀態變更為「已拒絕」（WR-003），系統通知申請人含拒絕原因（WR-007）
- 後置條件：假單狀態變更為「已核准」或「已拒絕」，申請人收到通知
- 關聯需求：FR-003, BR-017, FR-010

## UC-003: 員工撤回待審核假單

- 主要角色：員工（ROLE-001）
- 前置條件：員工有一筆自己的假單，狀態為「待審核」
- 觸發條件：員工選擇撤回該假單
- 基本流程：
  1. 員工在假單列表中選擇待審核的假單
  2. 員工選擇「撤回」
  3. 系統將假單狀態變更為「已撤回」（WR-004）
- 替代流程：
  - 1a. 假單狀態非「待審核」：系統不顯示撤回選項（PR-002）
- 後置條件：假單狀態變更為「已撤回」
- 關聯需求：FR-005, PR-002

## UC-004: 系統催辦逾期未審核假單

- 主要角色：系統（自動排程）
- 前置條件：存在狀態為「待審核」且自申請送出起已超過 3 天的假單
- 觸發條件：系統排程於每日 0:00 與 12:00 執行（NFR-003）
- 基本流程：
  1. 系統掃描所有狀態為「待審核」的假單
  2. 篩選出自申請送出時間起已超過 3 天的假單
  3. 對每筆符合條件的假單，發送 Email 通知給審核主管
  4. 同時發送站內訊息給審核主管
- 替代流程：
  - 2a. 無符合條件的假單：排程正常結束，不發送通知
  - 3a. 若主管已設代理人且在代理期間：通知發送給代理主管
- 後置條件：逾期假單的審核主管收到催辦通知
- 關聯需求：FR-004, WR-005

- 替代流程：無

- 關聯需求：FR-006, PR-006

## UC-006: HR 匯出 Excel 報表

- 主要角色：HR（ROLE-003）
- 前置條件：使用者具有 HR 角色
- 觸發條件：HR 點擊匯出功能
- 基本流程：
  1. HR 可選擇性設定部門篩選條件
  2. HR 可選擇性設定日期區間篩選條件
  3. HR 點擊匯出
  4. 系統產生 Excel 檔案，包含欄位：員工姓名、部門、假別、開始日期、結束日期、時數、申請日期、審核狀態、審核人
  5. 時數以 8 小時 = 1 天換算（BR-023）
- 替代流程：
  - 1a/2a. 未設定篩選條件：匯出全部紀錄
- 後置條件：HR 取得 Excel 報表檔案
- 關聯需求：FR-007, DR-004, DR-005, BR-023

## UC-007: 主管指定代理主管

- 主要角色：主管（ROLE-002）
- 前置條件：使用者具有主管角色
- 觸發條件：主管進入代理設定頁面
- 基本流程：
  1. 主管進入代理主管設定頁面
  2. 主管選擇代理人
  3. 主管填寫代理起訖日期
  4. 系統儲存代理設定
- 替代流程：
  - 2a. 主管取消現有代理設定：系統移除代理，回歸原主管審核（BR-020）
- 後置條件：代理設定已儲存，代理期間內假單將自動指派給代理主管
- 關聯需求：FR-008, BR-020, PR-004

## UC-008: 代理主管審核假單

- 主要角色：代理主管（ROLE-004）
- 前置條件：代理主管在代理期間內，有待審核假單屬於被代理主管的下屬
- 觸發條件：代理主管開啟待審核假單
- 基本流程：
  1. 代理主管查看假單詳情
  2. 代理主管選擇「同意」或「拒絕」（同 UC-002 基本流程）
  3. 審核結果為最終結果，原主管不需再次確認（BR-026）
  4. 系統通知申請人審核結果（FR-010）
- 替代流程：同 UC-002 替代流程
- 後置條件：假單審核完成，結果為最終
- 關聯需求：FR-003, FR-008, BR-026, PR-005
