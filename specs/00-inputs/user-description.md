# User Description

## 原始描述

我們公司大概有一百多個人，現在請假都是用 Line 群組跟主管說，然後主管再口頭通知 HR。很亂，常常忘記或算錯。我們想做一個系統，系統平台為 Web，讓員工可以線上申請請假，主管可以在上面審核，HR 可以看到所有人的紀錄。
假別的話，我們有特休、病假、事假、喪假。特休每年一開始就給，年資不同天數不同，這個我之後可以給你 Excel。病假一年三十天，事假一年十四天，喪假看親等。這些都是法定的，應該不會差太多，每年重置日期由到職日期決定。
申請的時候要填開始日期、結束日期、假別、還有原因。病假超過 3 天（含）才需要上傳診斷書，3 天以下不強制，有些同仁可能就是一般感冒。
主管審核就是同意或拒絕，拒絕要寫原因。如果主管超過三天沒有回，我們希望系統可以催一下，做 Email 通知跟系統通知，每天的0點跟12點都通知。
待審核的假單可以撤回。已經核准的不能自己取消，要重新申請一張銷假單讓主管審核，但銷假功能可以放第二期再做，第一期先不做。
HR 的話，他們要能看到所有人的假期紀錄，報表先做一種就好，Excel 格式匯出，欄位要有：員工姓名、部門、假別、開始日期、結束日期、時數、申請日期、審核狀態、審核人。可以用部門和日期區間當篩選條件。
代理主管由主管自己在系統上指定，要設定代理起迄日期。主管沒有設代理人的話，就維持原本的主管審核，不自動指派。HR 可以在後台看到誰設了代理人。
