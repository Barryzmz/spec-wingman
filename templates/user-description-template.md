# User Description Template

使用說明：將此模板複製至 `specs/00-inputs/user-description.md`，依各區塊填入資訊。
填寫完畢後執行 `prompts/01-read-inputs.prompt.md`。

填寫原則：
- 只填你確實知道的內容。不確定的事不要猜，直接留空或寫「尚未確認」。
- 不用寫得很正式。口語描述、會議紀錄、逐字稿都可以貼在「原始描述」區塊。
- 你寫得越清楚，AI 需要問的問題越少。

---

## 原始描述

> 在此貼上使用者口頭需求、訪談逐字稿、會議摘要、簡報重點或任何原始敘述。
> 可以是口語、條列或段落，不需要整理成正式格式。
> 若有多次訪談或多個來源，請分段並標記是哪一次的內容（例如：【第一次訪談 2026-05-31】）。

## 附件與來源

> 列出你提供的所有資料來源。每個來源給一個 SRC-ID，AI 在萃取事實時會引用這些 ID。

| 來源 ID | 類型 | 名稱 | 位置 | 備註 |
| --- | --- | --- | --- | --- |
| SRC-001 | （口頭訪談 / 會議紀錄 / Excel / Word / PDF / 簡報） | （來源名稱） | （本文件 / 檔案路徑 / 雲端連結） | （日期、提供者、是否已書面確認） |
| SRC-002 | 待提供 | （尚未交付的資料名稱） | （預計提供方式） | （預計提供時間） |

來源類型說明：
- **口頭訪談**：和 PM、使用者或利害關係人的對話記錄
- **會議紀錄**：正式或非正式的會議摘要
- **Excel / Word / PDF**：已有的規格文件、流程圖、資料表
- **簡報**：產品規劃或需求說明的投影片
- **待提供**：知道有這份資料但尚未拿到

## 填寫檢查清單

執行 Prompt 01 前，確認以下項目：

- [ ] 原始描述已填入（至少有一段口述或摘要）
- [ ] 所有來源都有 SRC-ID
- [ ] 待提供的資料已在來源清單中標記為「待提供」
- [ ] 不確定的事已列在「尚未確認的事」區塊（不要把不確定的內容混在確定的功能裡）
- [ ] 明確不做的事已列出（若有的話）
